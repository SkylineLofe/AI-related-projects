# AI-related-projects
Projects I've done in relation to learning how to build my own Gen AI models

The first few projects will be in HTML as I can't seem to find the actual source file. If I find them, I shall add the actual ipynb file.


In the first project (FoodHub), the goal was to perform an exploratory data analysis and provide actionable insights for a food aggregator company to get a fair idea about the demand of different restaurants and cuisines, which would help them enhance their customer experience and improve the business. This could have been improved by deepening my multivariate analysis with additional plots (for example, I could have added in point-points), and I could have cleaned up any lingering error outputs that existed while I was working and completing the project. 

In the second project (Personal Loan Campaign), the goal was to to identify bank customers with a high likelihood of purchasing a loan. In order to do this, I had to analyze the provided data to understand key customer attributes influencing loan acquisition. In doing so, I built a predictive model that captured patterns and customer characteristics, which would help the bank effectively target potential loan buyers, improving marketing efforts and increasing conversion rates. For this project, I felt like I did a fairly good job at my first attempt. I do plan on going back once again to do a second try, with the knowledge I now have, to see if I could improve this any further. 

For the third project (EasyVisa), the goal was to analyze the data of visa applicants, build a predictive model to facilitate the process of visa approvals, and based on important factors that significantly influence the visa status recommend a suitable profile for the applicants for whom the visa should be certified or denied. While working on the project and a little after I finished it, I felt I had successfully completed the project in a strong manner. However, after taking a very brief look back, I noticed that I could have provided more detailed justifications for my technical choices and I could have more explicitly linked my insights back to the model's performance metrics. This allowed me to realize that there definitely could be improvements to the code, either to make it more efficient or to challenge myself in completing this project in a different manner.
